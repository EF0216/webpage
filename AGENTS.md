# AGENTS.md

## このリポジトリの目的

このリポジトリは、商品LPをHTML/CSSで作成し、GitHub Pagesで公開するための作業リポジトリです。

主な用途は以下です。

- 商品LPの作成
- 既存LPの文言修正
- 画像差し替え
- デザイン調整
- GitHub Pagesでの公開管理
- Codex / Claude Code による継続修正

## 基本方針

このリポジトリでは、既存のデザインを大きく崩さず、小さな修正を優先してください。

商品LPは「上品・余白・ミニマル・商品がきれいに見える」ことを重視します。

参考トーンは、AURALEEのような余白のあるシンプルな商品ページです。

## フォルダ構成ルール

LPは商品・シーズンごとにフォルダを分けて管理します。

```text
webpage
├─ index.html
├─ AGENTS.md
├─ DESIGN_RULES.md
├─ README.md
├─ RIRALEGI-AW
│  ├─ index.html
│  └─ assets
└─ RIRALEGI-SS
   ├─ index.html
   └─ assets
```

各LPフォルダには、必ず以下を入れてください。

```text
index.html
assets/
```

画像は必ず各LPフォルダ内の `assets` フォルダに保存してください。

## 画像パスのルール

画像パスは、必ず相対パスで書いてください。

正しい例：

```html
<img src="assets/hero.png" alt="着用イメージ">
```

避ける例：

```html
<img src="/assets/hero.png">
<img src="C:/Users/.../hero.png">
```

ローカルPCの絶対パスは使わないでください。

## HTML/CSSのルール

基本は、1ページ完結型のHTMLにしてください。

各LPの `index.html` の中に、必要なCSSを `<style>` として記述します。

外部CSSファイルを増やす場合は、事前に理由があるときだけにしてください。

基本構成：

```text
index.html
assets/
```

を優先します。

## 修正方針

修正は、できるだけ小さく行ってください。

以下を守ってください。

1. 既存LPの構成を勝手に大きく変更しない
2. 画像ファイル名を勝手に変更しない
3. フォルダ名を勝手に変更しない
4. 画像パスを壊さない
5. デザインの方向性を大きく変えない
6. スマホ表示が崩れないようにする
7. 変更理由がわかるように説明を残す
8. 既存の公開URLを壊さない

## LPに必要な基本セクション

商品LPは、基本的に以下の流れを推奨します。

1. HERO
2. 商品特徴
3. LINE UP
4. SCENE
5. TEXTURE
6. SPEC

ただし、商品特性に合わせて不要なセクションは省略しても構いません。

## デザインの優先順位

優先順位は以下です。

1. 商品がきれいに見える
2. 余白があり、上品に見える
3. 文字が読みやすい
4. スマホでも崩れない
5. 画像パスが管理しやすい
6. 今後アイテム追加しやすい

## 禁止事項

以下は禁止です。

- 画像を外部URLに依存させる
- ローカルPCの絶対パスを使う
- 既存画像名を勝手に変更する
- assetsフォルダ以外に画像を散らす
- 派手な色や過度な装飾を追加する
- 既存LPの構成を大きく変更する
- GitHub Pagesで表示できない構成にする
- 必要以上にJavaScriptを追加する

## 公開URLの考え方

GitHub Pagesでは、各LPフォルダの `index.html` がページになります。

例：

```text
https://ユーザー名.github.io/webpage/RIRALEGI-AW/
https://ユーザー名.github.io/webpage/RIRALEGI-SS/
```

トップの `index.html` は、LP一覧ページとして使います。

## 作業時の確認項目

修正後は、必ず以下を確認してください。

- トップページが表示されるか
- 各LPページが表示されるか
- 画像がすべて表示されるか
- PC表示で崩れていないか
- スマホ表示で大きく崩れていないか
- リンク先が正しいか
- GitHub Pages上でも表示されるか

## このリポジトリで重視すること

このリポジトリでは、HTMLの高度な技術よりも、商品企画や売場提案に使いやすいLPを素早く作ることを重視します。

そのため、複雑な構成よりも、わかりやすく修正しやすい構成を優先してください。
