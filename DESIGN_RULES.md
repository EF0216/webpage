# DESIGN_RULES.md

## デザインの基本思想

このリポジトリのLPは、商品をきれいに見せることを最優先にします。

デザインの方向性は以下です。

- 上品
- 余白がある
- ミニマル
- 商品写真が主役
- 文字は控えめだが読みやすい
- 派手すぎない
- 売場提案や商談資料にも使いやすい

参考トーンは、AURALEEのような余白のある商品ページです。

## 色の基本

基本カラーは以下を推奨します。

```text
背景：#f8f6f1 / #fbfaf6 / #fffdf8
文字：#20201d / #25231f
サブ文字：#706a60 / #777970
線：#e0d8cc / #deded7
AW系：#efe4d5 / #e7e1dc
SS系：#eef5ee / #f6faf6
```

強い原色は避けてください。

特に以下は原則使いません。

- 真っ赤
- 原色の青
- 強い黄色
- 蛍光色
- 派手なグラデーション

## 文字・タイポグラフィ

見出しは、明朝体・セリフ系の上品な印象を優先します。

推奨：

```css
font-family: Georgia, "Times New Roman", "Yu Mincho", serif;
```

本文は読みやすさを優先します。

推奨：

```css
font-family: "Helvetica Neue", Arial, "Hiragino Sans", "Yu Gothic", Meiryo, sans-serif;
```

## レイアウト方針

基本は以下です。

- 余白を広めに取る
- セクションごとに空気感を出す
- 画像を大きく使う
- 文字は詰め込みすぎない
- 1画面に情報を詰め込みすぎない
- 商品写真・着用画像・素材画像の順で訴求する

## LP基本構成

推奨構成は以下です。

```text
1. HERO
2. FEATURES
3. LINE UP
4. SCENE
5. TEXTURE
6. SPEC
```

### HERO

目的：

- 最初に商品の世界観を伝える
- 商品の一番強いベネフィットを伝える
- 価格を見せる

例：

```text
ラクなのに、きれい見え。
秋から冬まで、心地よく。
本体価格 799円 / 税込価格 878円
```

### FEATURES

目的：

- 商品特徴を短く整理する

例：

```text
すっきり見える
やさしい微起毛
冬まであたたかい
選べる素材感
```

### LINE UP

目的：

- 商品展開を見せる
- シーズン別や素材別に分ける

例：

```text
AUTUMN：9月・10月・11月 プリーツ微起毛
WINTER：12月・1月・2月 リブ・ベロア裏起毛
```

### SCENE

目的：

- 実際に使う生活シーンを伝える

例：

```text
おうち時間に
ちょっとしたお出かけに
旅行やレジャーに
冷え込む日にもおすすめ
```

### TEXTURE

目的：

- 素材の良さを伝える
- 触感・季節感・機能性を補足する

例：

```text
プリーツ微起毛
リブセーター風
ベロア
```

### SPEC

目的：

- 最後に商品情報を整理する

例：

```text
価格
サイズ
展開時期
素材タイプ
訴求ポイント
```

## 画像ルール

画像は各LPフォルダの `assets` に保存します。

例：

```text
RIRALEGI-AW
├─ index.html
└─ assets
   ├─ hero.png
   ├─ pleats_series.png
   ├─ winter_series.png
   ├─ scene_aw.png
   └─ texture_aw.png
```

画像名は英数字・アンダースコア・ハイフンのみを推奨します。

良い例：

```text
hero.png
scene_aw.png
texture_aw.png
winter_series.png
```

避ける例：

```text
秋冬 着用画像.png
新しい画像 (1).png
C画像最終版.png
```

## 画像の見せ方

商品画像：

- 背景は白・薄いグレー・生成り
- 商品が中央に見えること
- 余白を取りすぎない
- 商品同士の大きさを揃える

着用画像：

- 自然光
- 明るい室内
- ベージュ・白・生成り系
- 商品のシルエットが見える
- 顔よりも着用感が伝わることを優先

素材画像：

- 1枚画像でまとめてもよい
- 3素材を比較しやすく見せる
- 文字入り画像を使う場合は、HTML側で文字を重ねすぎない

## 価格表記ルール

価格は以下を基本とします。

```text
本体価格 799円
税込価格 878円
```

HTML内では、価格ボックスとして表示することを推奨します。

## スマホ対応ルール

スマホでは以下を守ります。

- 横2列は1列にする
- 文字サイズを詰め込みすぎない
- 画像が画面幅からはみ出さない
- ナビゲーションは非表示でもよい
- 価格ボックスが横にはみ出さないようにする

## してはいけないデザイン

以下は避けてください。

- 文字を詰め込みすぎる
- 背景色を濃くしすぎる
- 強い影を多用する
- 派手なアニメーションを入れる
- 画像の縦横比を無理に潰す
- 商品画像の大きさが左右でバラバラになる
- LP全体のトーンをページごとに変えすぎる

## 今後アイテム追加時のルール

新しいLPを追加するときは、以下の形式でフォルダを増やします。

```text
CHARACTER-SOCKS
MENS-SCHOOL-SOCKS
ROOMWEAR-AW
LEGWEAR-SS
```

各フォルダには必ず以下を入れます。

```text
index.html
assets/
```

トップの `index.html` には、商品一覧カードを追加します。
